Abstract

The SFI Group System is the most widely used classification system for the maritime and offshore industry worldwide. It is an international standard, which provides a functional subdivision of technical and financial ship or rig information. SFI consists of a technical account structure covering all aspects of ship/rig specification, and it can be used as a basic standard for all systems in the shipping/offshore industry. More than 6000 SFI systems have been installed worldwide. SFI is used by shipping and offshore companies, shipyards, consultancies, software suppliers, authorities and classification societies.

Example

Here is a very basic document describing a detail code:

sdir:641039_Salinometers
        rdfs:label       "Salinometers"@en ;
        rdfs:subClassOf  sdir:DetailCode , sdir:641_MAIN_BOILERS ;
        sdir:code        "641.039" ;
        sdir:definition  "A salinometer is a device designed to measure the salinity, or dissolved salt content,..." .

This brief example shows how a detail code is described using RDF. It shows which group it is sub class of, which again shows where in the hierarchy it is located. It contains the property code which is the unique "ID" and the property definiton which describes the detail code further.

The Semantic Web

To a computer, the Web is a flat, boring world, devoid of meaning. This is a pity, as in fact documents on the Web describe real objects and imaginary concepts, and give particular relationships between them. For example, a document might describe a person. The title document to a house describes a house and also the ownership relation with a person. Adding semantics to the Web involves two things: allowing documents which have information in machine-readable forms, and allowing links to be created with relationship values. Only when we have this extra level of semantics will we be able to use computer power to help us exploit the information to a greater extent than our own reading. - Tim Berners-Lee "W3 future directions" keynote, 1st World Wide Web Conference Geneva, May 1994

The modelling of the SFI Coding and Classification System is an attempt to digitalize the model and automate the usage of the classification system.

Classes

Class: sdir:SFIConcept

SFIConcept - The root concept of the ontology. Every group is sub class of this class.
Status: stable
Properties include: code definition
Used with:

The SFIConcept class is the most basic concept in the SFI domain.

[#] [back to top]


Class: sdir:MainGroup

MainGroup - The top group in the hierarchy, containing 8 main groups.
Status: stable
Properties include: code definition
Used with:

The MainGroup class is the top group in the hierarchy, containing the groups; 1 Ship General, 2 Hull, 3 Equipment For Cargo, 4 Ship Equipment, 5 Equipment For Crew and Passengers, 6 Machinery Main Components, 7 System For Machinery Main Components, 8 Ship Common Systems.

[#] [back to top]


Class: sdir:Group

Group - The sub group of MainGroup in the hierarchy, containing several groups.
Status: stable
Properties include: code definition
Used with:

The Group class is the sub group of MainGroup.

[#] [back to top]


Class: sdir:SubGroup

SubGroup - The sub group of Group in the hierarchy, containing several sub groups.
Status: stable
Properties include: code definition
Used with:

The SubGroup class is the sub group of Group.

[#] [back to top]


Class: sdir:DetailCode

DetailCode - DetailCode is the lowest representation, detailing specific components.
Status: stable
Properties include: code definition
Used with:

The DetailCode class details specific components.

[#] [back to top]


Properties

Property: sdir:code

code - code for the entity.
Status: stable
Domain: having this property implies being a SFIConcept

The code property is a textual string of the unique code.

[#] [back to top]


Property: sdir:definition

definition - definition of the entity.
Status: stable
Domain: having this property implies being a SFIConcept

The definition property is a textual string describing the entity.
